# 001 — Красивая документация на GitHub Pages

## Задача
Сделать документацию проекта AnKor Shell в виде красивого сайта на GitHub Pages.

## Желаемый результат
- Опубликованный сайт документации на GitHub Pages.
- Приятный, аккуратный внешний вид (тема/оформление).
- Документация покрывает публичный API библиотеки (модули `sdk/*`, функции `ak.<domain>.<function>()`, CaLS).

## Контекст / зацепки
- Проект — модульная Bash/Zsh библиотека, грузится через `index.sh`; нет build-шага и генератора доков.
- Функции документированы JSDoc-стилем (`@param`, `@output`, `@example`) прямо в `.sh`.
- В `sdk/doc.sh` уже есть documentation-утилиты — возможная зацепка для генерации.
- Структура модулей и префиксы доменов описаны в `CLAUDE.md`.

## Открытые вопросы
- Генератор/движок: статический (MkDocs Material / Docusaurus / Just the Docs) или что-то Bash-специфичное?
- Автогенерация из JSDoc-комментариев в `.sh` vs ручные Markdown-страницы?
- Деплой: GitHub Actions workflow или ветка `gh-pages`?
- Насколько «красиво» — кастомная тема/брендинг или готовая тема из коробки?

> Детали движка и оформления решаются на этапе `/flow:design`.
